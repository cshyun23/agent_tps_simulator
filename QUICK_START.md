# Quick Start - 5분 안에 시작하기

Agent TPS Simulator를 처음 사용하시나요? 이 가이드는 **5분 안에 첫 시뮬레이션**을 완료할 수 있도록 합니다.

---

## ⚡ 초고속 세팅 (1분)

### 1. 서버 확인

두 개의 터미널을 열고 각각 실행:

**터미널 1 (백엔드):**
```bash
cd backend
python main.py
# 출력: "Uvicorn running on http://127.0.0.1:8000"
```

**터미널 2 (프론트엔드):**
```bash
cd frontend
npm run dev
# 출력: "Local: http://localhost:5173"
```

### 2. 브라우저 열기

```
http://localhost:5173
```

**완료!** ✅

---

## 🔷 Step 1: LLM 서버 생성 (1분)

1. 상단 탭에서 **`LLM Hub`** 클릭
2. **`+ 서버 추가`** 버튼
3. 아래 값 입력:

```
이름: Claude Opus
모델: Claude-3.5
파라미터: 70
GPU 종류: h100-80g
GPU 개수: 4
Ref TTFT: 85
Ref TPOP: 18
```

4. **`저장`** 버튼

✅ 서버가 생성됨

---

## 🔵 Step 2: 플로우 선택 (30초)

1. **`Agent Flow`** 탭 클릭
2. 왼쪽 **`+ 새 플로우`** 버튼
3. **`단순 RAG`** 선택
4. **`생성`** 클릭

```
생성된 구조:
[START] → [LLM: 질문 분석] → [TOOL: 검색] → [LLM: 답변] → [END]
```

✅ 플로우 생성 완료

---

## 🟡 Step 3: 노드 설정 (1분)

**LLM 노드 설정:**

1. 첫 번째 LLM 노드 클릭
2. 오른쪽 팝업에서:
   - **LLM 서버**: "Claude Opus" 선택
   - **입력 토큰**: 512
   - **출력 토큰**: 128
3. **닫기**

4. 두 번째 LLM 노드도 동일하게 설정

**Tool 노드 설정:**

1. Tool 노드 클릭
2. **처리 시간**: 500 (ms)
3. **닫기**

✅ 모든 노드 설정 완료

---

## 🟢 Step 4: 시뮬레이션 실행 (1.5분)

### 설정

우측 "Simulation Panel"에서:

```
도착 패턴: Ramp-up
  Peak Users: 100
  Ramp Duration: 10
  Hold Duration: 30
  Ramp Shape: Linear

총 시간: 60초

재생 속도: 10x

P95 메트릭: ☑️ 체크
P99 메트릭: ☑️ 체크
```

### 실행

1. 상단 우측 **`▶ Run Simulation`** 클릭
2. 자동으로 시뮬레이션 시작
3. 차트 시청 (약 30초)

```
⏳ 시뮬레이션 진행 중...
재생속도: 10x
```

✅ 완료!

---

## 📊 결과 확인 (1분)

시뮬레이션 완료 후 우측 하단:

```
📊 요약
─────────────────
처리됨: 3,000개
실패: 5개 (0.2%)

E2E Latency: 248ms
Peak TPS: 52.1

P95: 352ms
P99: 451ms
```

### 결과 해석

| 메트릭 | 값 | 의미 |
|--------|-----|------|
| **처리됨** | 3,000 | ✅ 좋음 (많이 처리) |
| **실패** | 5개 | ✅ 좋음 (거의 없음) |
| **TPS** | 52.1 | ✅ 좋음 (충분한 처리량) |
| **E2E** | 248ms | ✅ 좋음 (빠른 응답) |
| **P99** | 451ms | ✅ 좋음 (최악의 경우도 괜찮음) |

💡 **결론**: 이 구성으로는 100명 동시 사용자를 충분히 처리할 수 있습니다!

## 🎯 다음 단계

### 성능 개선 테스트

1. **다른 구성 테스트**:
   - Peak Users를 150으로 증가
   - Ramp Duration을 5초로 단축
   - 다시 시뮬레이션

2. **결과 비교**:
   - **비교 뷰** 탭
   - 두 결과 체크박스 선택
   - **`비교 (2 선택)`** 클릭
   - 어떤 구성이 더 좋은지 확인

3. **최적화**:
   - 필요한 부분 수정
   - 반복 테스트
   - 최고 성능 찾기

### 플로우 커스터마이징

현재 RAG 플로우를 수정하여:
- 더 많은 노드 추가
- 조건부 분기 추가
- 병렬 처리 추가

**자세한 방법**: `USER_MANUAL.md` 참고

## ❓ 빠른 문제 해결

### "서버를 찾을 수 없습니다" 에러

**해결:**
```bash
# 백엔드가 실행 중인지 확인
curl http://localhost:8000/health

# 실행 안 되면 시작
cd backend
python main.py
```

### 시뮬레이션이 시작 안 됨

**확인사항:**
- ✅ Start 노드와 End 노드가 있는가?
- ✅ LLM 서버를 선택했는가?
- ✅ 플로우를 저장했는가? (상단 `💾 Save`)

### 결과가 비정상적

**확인:**
1. LLM 파라미터 (TTFT, TPOP) 재확인
2. 토큰 수 재확인
3. 브라우저 개발자 도구 (F12) → Console 에러 확인
