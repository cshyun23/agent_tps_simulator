# 사용 가이드

## 개요

Agent TPS Simulator는 3가지 주요 화면으로 구성됩니다:

1. **Agent Flow** - 플로우 설계 및 시뮬레이션
2. **LLM Hub** - 가상 LLM 서버 관리
3. **비교 뷰** - 시뮬레이션 결과 비교

---

## 1. Agent Flow 탭

### 1-1. 플로우 생성

1. 왼쪽 사이드바에서 `+ 새 플로우` 버튼 클릭
2. 템플릿 선택 모달에서 원하는 템플릿 선택:
   - **빈 플로우**: 처음부터 직접 작성
   - **단순 RAG**: Start → LLM → Tool → LLM → End
   - **ReAct 루프**: Conditional 분기가 포함된 템플릿
   - **병렬 Tool 호출**: Parallel 노드가 포함된 템플릿

### 1-2. 플로우 편집

#### 노드 추가

상단 왼쪽 버튼으로 노드 추가:
- `+ Start` - 시작 노드 (필수 1개)
- `+ LLM` - LLM 추론 노드
- `+ Tool` - 외부 도구 호출 노드
- `+ Cond` - 조건부 분기 노드
- `+ Para` - 병렬 실행 노드
- `+ End` - 종료 노드 (필수 1개 이상)

#### 노드 연결

1. 노드의 동그란 핸들을 드래그하여 다른 노드에 연결
2. 루프백 (같은 노드 또는 이전 노드로 연결)은 점선으로 표시됨

#### 노드 설정

1. 노드 **클릭**하면 오른쪽 패널에 설정 팝업 표시
2. 각 노드 타입별 설정값 입력:
   - **LLM 노드**: 서버 선택, 입력/출력 토큰 수, 최대 동시 처리 수
   - **Tool 노드**: 처리 시간(ms)
   - **Conditional 노드**: 분기 목록, 확률(%), 실패율, 재시도 설정
   - **Parallel 노드**: (자동 처리)

#### Conditional 분기 설정

1. Conditional 노드 클릭
2. "분기 목록" 섹션에서 `+ 추가` 버튼으로 분기 추가
3. 각 분기별 설정:
   - **분기 이름**: 이 분기의 이름 (예: "의도 명확", "추가 질문 필요")
   - **확률(%)**: 0~100, 모든 분기의 합이 100% 필요
   - **대상 노드**: 다음 노드의 ID

#### 플로우 저장

상단 중앙 `💾 Save` 버튼으로 저장. 모든 변경사항이 백엔드에 저장됨.

#### 플로우 관리

사이드바의 플로우 항목에서:
- **클릭**: 플로우 선택 (파란색 하이라이트)
- **더블클릭**: 플로우 이름 인라인 편집
- **Hover 후 버튼 클릭**: 
  - `📋 복제` - 플로우 복제
  - `🗑 삭제` - 플로우 삭제

#### Import/Export

- `📤 Export` - 현재 플로우를 JSON 파일로 다운로드
- `📥 Import` - 이전에 저장한 JSON 파일에서 플로우 로드

### 1-3. 시뮬레이션 실행

#### 설정

우측 패널 "Simulation Panel"에서 설정:

**기본 설정:**
- **도착 패턴** (Arrival Pattern):
  - **Ramp-up**: 점진적 증가
    - Peak Users: 최대 동시 사용자 수
    - Ramp Duration: 증가 시간(초)
    - Hold Duration: 유지 시간(초)
    - Ramp Shape: Linear 또는 Smooth
  - **Wave**: 사인파 패턴
    - Min Users: 최소 사용자
    - Peak Users: 최대 사용자
    - Period: 주기(초)
- **총 시간** (초): 시뮬레이션 총 진행 시간
- **재생 속도**: 1x, 2x, 5x, 10x (빠른 테스트용)
- **P95/P99 메트릭 표시**: 체크하면 차트에 백분위수 표시

#### 실행

1. 플로우 에디터에서 Start 노드와 End 노드 선택
2. 상단 우측 `▶ Run Simulation` 버튼 클릭
3. 설정 후 시작 버튼으로 시뮬레이션 실행

#### 실시간 모니터링

시뮬레이션 중:
- **실시간 차트**: TPS, E2E Latency, P95, P99 메트릭 표시
- **재생 속도**: 빠르게 테스트하려면 높은 배속 선택
- **일시정지/재개**: `⏸ 일시정지` 버튼으로 일시정지 가능
- **초기화**: `⏹ 초기화` 버튼으로 시뮬레이션 중단

#### 결과 요약

시뮬레이션 완료 후:

| 메트릭 | 설명 |
|--------|------|
| 처리됨 | 성공적으로 처리된 요청 수 |
| 실패 | 실패한 요청 수 및 실패율(%) |
| E2E Latency | 평균 엔드-투-엔드 지연시간(ms) |
| Peak TPS | 최고 처리량(요청/초) |
| P95, P99 | 95%, 99% 백분위수 지연시간(ms) |

#### 결과 저장

- `📥 JSON` - 전체 결과를 JSON 파일로 다운로드
- `📥 CSV` - 시간대별 스냅샷을 CSV로 다운로드 (Excel 분석용)

## 2. LLM Hub 탭

### 2-1. 가상 LLM 서버 생성

1. `+ 서버 추가` 버튼 클릭
2. 서버 정보 입력:

| 필드 | 설명 | 예시 |
|------|------|------|
| 이름 | 서버 식별명 | "Claude Opus" |
| 모델 | 모델 이름 | "Claude-3.5-Sonnet" |
| 파라미터 (B) | 모델 파라미터 크기 | 70 (70B Llama-3 의미) |
| GPU 종류 | GPU 모델 | "h100-80g" |
| GPU 개수 | 보유 GPU 수 | 4 |
| Ref TTFT (ms) | 참고 TTFT 지연시간 | 85 (512 토큰 기준) |
| Ref TPOP (ms/tok) | 참고 TPOP (토큰당 생성 시간) | 18 |

3. `저장` 버튼으로 서버 생성

### 2-2. 서버 관리

각 서버 카드의 버튼:
- `✏ 편집` - 서버 정보 수정
- `🗑 삭제` - 서버 삭제

### 2-3. GPU 레퍼런스 관리

하단 "GPU 레퍼런스" 섹션:
- `📤 Export` - GPU 레퍼런스 데이터 다운로드
- `📥 Import` - 저장된 GPU 레퍼런스 파일 업로드

## 3. 비교 뷰 탭

### 3-1. 시뮬레이션 결과 비교

#### 결과 선택

왼쪽 "시뮬레이션 히스토리"에서:
- **체크박스**: 비교할 결과 선택 (최대 제한 없음)
- **날짜**: 각 시뮬레이션 실행 날짜 표시

#### 비교 실행

1. 비교할 결과 2개 이상 선택
2. `비교 (선택 수)` 버튼 클릭
3. 아래 다양한 차트로 비교 분석

#### 비교 차트

| 차트 | 설명 |
|------|------|
| 요청 처리 | 결과별 완료/실패 요청 수 비교 |
| TPS 비교 | Peak TPS 비교 |
| Latency 비교 | E2E 평균 및 P95 지연시간 비교 |
| 백분위수 | P95 vs P99 비교 |

#### 요약 테이블

각 시뮬레이션의 핵심 메트릭을 한눈에 비교:
- Completed: 완료된 요청
- Failed: 실패한 요청
- E2E (ms): 평균 지연시간
- Peak TPS: 최고 처리량
- P95, P99: 백분위수 지연시간

#### 결과 삭제

히스토리 항목 우측 `🗑` 버튼으로 해당 결과 삭제

## 일반 팁

### 성능 분석 팁

1. **단일 구성 테스트**: 먼저 하나의 구성으로 기본값 측정
2. **변수 하나씩 변경**: 도착 패턴, 토큰 수, LLM 서버 등을 하나씩 변경하며 테스트
3. **재생 속도 활용**: 초기 테스트는 높은 배속으로 빠르게 확인
4. **P95/P99 주목**: 평균만 좋아도 높은 백분위수가 나쁜 경우가 있음

### 플로우 설계 팁

1. **Start/End 필수**: 시뮬레이션 전에 Start 노드와 End 노드 선택 필요
2. **분기 확률 합 100%**: Conditional 노드의 분기 확률 합이 정확히 100%여야 함
3. **루프 제한**: 루프백은 가능하지만, 최대 10번 반복으로 제한
4. **연결 검증**: 모든 노드가 Start에서 End로 도달 가능해야 함

## 문제 해결

### 시뮬레이션이 시작되지 않음

- **확인 사항**:
  - Start 노드와 End 노드 선택 여부
  - 플로우 저장 여부
  - 백엔드 서버 실행 여부 (localhost:8000)

### 결과가 비정상적임

- **확인 사항**:
  - LLM 서버 설정 (TTFT, TPOP 값)
  - 토큰 수 설정 (입력/출력 토큰)
  - 도착 패턴 설정 (Ramp-up vs Wave)

### 플로우 로드/저장 실패

- **해결책**:
  - 브라우저 콘솔 확인 (F12)
  - 백엔드 로그 확인
  - 파일 용량 (매우 큰 플로우는 로드 시간 길음)
